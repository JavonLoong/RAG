# 动力装备知识库前端重设计 Image Prompt

## 目标

为中文 RAG/GraphRAG 前端控制台“动力装备知识库”生成一张高保真、可落地的产品界面主视觉图。这是工程工作台而不是营销页，适合 GitHub 项目展示、课程答辩和技术演示，整体风格专业、克制、信息密度高、真实可实现。

## 画面构成

- 单屏桌面端控制台，目标尺寸 `1536x1024`，内容完整、结构清晰、可直接映射为 `HTML/CSS/Vanilla JS`。
- 顶部固定导航栏包含：产品名“动力装备知识库”、项目状态、环境切换、用户入口、全局搜索、帮助、导出总览按钮。
- 主工作区采用三栏或双栏工程台布局，强调功能分区而非装饰分层，避免卡片套卡片。
- 左侧为数据与流程控制，中间为检索与问答主操作区，右侧为知识图谱与观测面板，底部或侧栏承载日志与评测信息。

## 必备模块

- 顶部导航：深浅中性背景、明确层级、含项目名、当前模式、数据集状态、运行状态、时间戳。
- 标准模式切换：`RAG / GraphRAG` 明显可点击的分段切换器，显示当前选中状态与切换说明。
- 上传入库：支持拖拽上传、批量文件列表、进度条、文件类型、大小、失败重试、入库按钮。
- 处理流程：`OCR -> 清洗 -> 分块 -> 向量化 -> 索引构建` 的线性流程条，展示每一步状态、耗时、成功/失败标记。
- 集合/来源表格：表格展示集合名、来源数量、更新时间、文档类型、索引状态、操作入口。
- 语义检索输入：大输入框，支持关键词、过滤条件、TopK、相似度阈值、时间范围。
- RAG 问答：对话区显示用户问题、系统回答、引用来源、置信度、可追问建议。
- 知识图谱预览：右侧小型图谱视图，节点与边清晰，支持实体高亮、关系路径、缩放控件。
- 评测指标：展示命中率、召回率、答案一致性、引用覆盖率、延迟、Token 消耗等。
- 对话记忆：显示当前会话的历史记忆摘要、保留轮次、可清空/固定记忆。
- Local session 状态：本地会话保存提示、未同步状态、自动恢复、最近操作记录。
- 导出按钮：支持导出问答记录、索引报告、图谱快照、评测结果，按钮明确且可见。
- 日志/可观测性：底部日志面板显示事件流、警告、耗时、队列状态、处理成功率，含时间戳与级别标签。

## 视觉风格

- 工业软件、数据平台、研发控制台风格，强调可读性、密度、秩序感。
- 使用中性灰、白、深蓝灰、少量橙色或青绿色作为状态强调色。
- 组件边框细、圆角小、阴影极轻，层级主要靠间距与排版建立。
- 字体为现代中文无衬线风格，数字和表格信息清晰，适合屏幕展示。
- 图表、表格、表单、开关、按钮、标签、状态点都要像真实产品，避免概念化。

## 版式细节

- 顶部导航高度适中，左侧产品标识，中部模式与搜索，右侧导出、设置、账号。
- 主区左栏放上传入库和流程状态，中栏放检索与问答、引用结果、表格化证据，右栏放图谱预览和评测指标。
- 底部固定日志区或右下日志抽屉，显示系统事件流与可观测性数据。
- 表格必须有真实列名和行数据，不要空表或占位图。
- 所有按钮、标签、输入框、进度条都应具备真实交互语义。
- 保持界面克制、工程感强、信息丰富但不拥挤。

## 明确禁止

- 不要营销页风格，不要大标题口号，不要宣传文案。
- 不要紫色赛博朋克、霓虹渐变、发光球体、玻璃拟态过强。
- 不要浏览器外壳、设备框、手机壳、夸张装饰。
- 不要抽象概念插画、人物、3D 夸张图形、空泛背景纹理。
- 不要卡片嵌套卡片，不要过度圆角，不要漂浮大阴影。
- 不要英文为主，界面文字以中文为主，必要时保留少量英文技术术语。

## 生成要求

- 画面必须像一个真实可开发的中文企业级前端控制台。
- 所有区域都有明确功能，不留大面积空白。
- 细节足够丰富，适合项目答辩截图与 GitHub README 首图。
- 界面应具有高可用性、逻辑清晰、可快速还原为前端页面。
