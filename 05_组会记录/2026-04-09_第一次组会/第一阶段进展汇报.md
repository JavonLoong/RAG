## 三、数据质量问题

### 问题1：极短文本块（7个，占比2.8%）

OCR识别过程中部分文本被错误切断，产生了不完整的碎片：

| 页码 | 标签 | 内容 | 原因分析 |
|------|------|------|---------|
| 第1页 | Para | "程体系。" | 句子被跨列切断 |
| 第1页 | Title | "建方法" | 标题被切断，完整应为"构建方法" |
| 第2页 | Para | "印证。" | 句末碎片 |
| 第2页 | Para | "统性。" | 句末碎片 |
| 第3页 | Para | "模型。" | 句末碎片 |

**根因**：原始PDF是双栏排版，OCR标注时部分文本块的bbox框切割过细，导致一个完整句子被拆成多个碎片。

**影响**：这些短碎片会被独立分块，降低检索质量（搜索"构建方法"可能匹配不到只有"建方法"的碎片）。

**建议**：
1. 在分块前合并极短块（<10字符）到相邻块
2. 或在标注阶段调大bbox范围，避免跨列切断

### 问题2：双栏文本交叉

检索结果中可以看到，部分chunk的内容出现了左右栏交叉的现象：

```
力支撑，为校企融合建设课程体系提供技术参考。    ← 左栏
关键词：产教融合；课程创新；...                   ← 左栏
船舶燃气轮机控制与健康管理课程是一门多学          ← 左栏
究如何建立健全...的课                              ← 右栏（突然跳到右栏）
科交叉融合的前沿课程...                           ← 左栏（又跳回来）
程体系。                                           ← 右栏碎片
```

**根因**：标注按纵坐标排序，但双栏PDF的左右栏纵坐标会重叠，导致排序后左右栏文本交叉。

**影响**：严重。检索到的段落内容不连贯，LLM无法正确理解上下文。

**建议**：
1. 在解析阶段加入横坐标（x坐标）判断，区分左栏和右栏
2. 先按栏分组（x<50%为左栏），再在每栏内按y坐标排序
