# 六个缺失项补齐验收报告

## 这次补了什么

1. 接 LLM：新增 OpenAI-compatible LLM adapter。没有 `OPENAI_API_KEY` 时会明确报错，不会用假回答顶替。

2. 图数据库：四本书三元组已导入本地 SQLite 图存储，同时生成 Neo4j Cypher 文件。当前没有 Neo4j 服务，所以没有声称 Neo4j 已实际运行。

3. GraphRAG 问答链路：新增最小 GraphRAG 编排，能把 ChromaDB 文本检索和图检索结果合并成上下文，再交给 LLM。当前已跑通 `context-only`，真实回答需要配置 LLM key。

4. LLM 三元组抽取：新增按 schema 调 LLM 抽取三元组的 pipeline。没有 key 时明确失败，不再把规则 baseline 说成 LLM 抽取。

5. 检索模块实代码：`retrieval_engine` 不再只是 README，已有 Chroma、Keyword、Hybrid 三类检索器。

6. 系统评测：新增 35 条评测集和评测脚本，不再只靠 10 个冒烟问题。
