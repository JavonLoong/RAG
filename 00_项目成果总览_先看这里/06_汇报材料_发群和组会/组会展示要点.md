## 汇报口径

这次我不是只做概念调研，也不是只做静态 HTML 展示，而是把之前的 JSON 文件接进了一个最小 Graph construction POC 工作台。流程是先从 Label Studio JSON 中抽出 OCR 文本块，再按标题和长度组织 chunk，然后用课程知识体系 schema 约束实体和关系，生成候选三元组，并为每条关系绑定 evidence。最后我可以在页面上逐条判断三元组对不对，并导出人工评审结果。

当前它证明的是：从真实 JSON 输入到图构建、证据展示、人工评审和结果导出的链路可以走通。人工评审后，27 条候选三元组中 26 条被判断为正确，1 条待讨论。它还没有证明 LLM 自动抽取质量、Neo4j 落库或完整 GraphRAG 问答已经完成。下一步会用同一个 schema 让 LLM 自动抽三元组，再和当前人工评审结果对比。
