# Layout-aware OCR 性质和风险清单

## 一句话结论

新版 Tesseract TSV layout-aware OCR 已经解决了“完全没有坐标、无法判断两栏顺序”的问题，但没有解决“文字识别一定正确”的问题。

它现在适合作为后续 RAG / GraphRAG 的候选语料底座，但不能直接当作精校文本。

## 这版解决了什么

- 保存了每一行的坐标框。
- 能标记页面的版面顺序风险。
- 对疑似两栏页面可以做阅读顺序重排。
- 后续可以按页码、栏号、坐标回到原 PDF 复核。

## 这版没有解决什么

- OCR 字符仍可能识别错。
- 公式、表格、图注仍可能进入正文。
- 页眉页脚仍可能混入语料。
- 高风险版面页仍需要人工抽检。
- 当前普通 RAG 的 ChromaDB 仍是旧 OCR 版，尚未重建为 layout-aware 版。

## 怎么汇报比较准确

可以说：

> 已经完成一版带坐标和版面顺序判断的 OCR。它比旧版更适合处理两栏 PDF，但质量上仍需抽检，尤其是高风险版面页、公式表格页和低置信度页。

不要说：

> OCR 已经完全准确。

也不要说：

> 现在可以直接拿 OCR 文本做最终知识图谱。

## 下一步建议

1. 从高风险页中抽 30 到 50 页人工检查。
2. 先挑 1 到 2 本书，用 layout-aware OCR 重建一个小型 ChromaDB。
3. 对比旧版 RapidOCR 和新版 Tesseract TSV 的检索效果。
4. 再决定是否用新版 OCR 全量重建 RAG 数据库。
