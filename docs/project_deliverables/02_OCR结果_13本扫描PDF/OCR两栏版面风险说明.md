# OCR 两栏版面风险说明

这个风险是真实存在的，而且比单纯“错几个字”更严重。

## 问题是什么

有些 PDF 页面是左右两栏。人读的时候是：

先读左栏从上到下，再读右栏从上到下。

但普通 OCR 可能按检测框顺序输出，变成：

左栏第一行 → 右栏第一行 → 左栏第二行 → 右栏第二行。

这样会让原本连续的一句话被拆开，也可能把两栏中无关的句子拼到一起。

## 对当前 OCR 结果的判断

当前全量 OCR 已经跑完，但旧结果只保存了文字行，没有保存每一行的坐标框。因此：

- 能证明：扫描 PDF 已经被转成可检索文本。
- 不能证明：句子和段落顺序完全符合原文。
- 不能证明：两栏页面的语义顺序是正确的。

所以现在这批 OCR 文本适合做普通 RAG 的候选检索库，不适合直接作为知识图谱抽取的干净语料。

## 对不同任务的影响

- 普通 RAG 粗检索：可以先用，因为检索主要看关键词和局部证据。
- 精确问答：需要回看原 PDF 页图，不能只信 OCR 文本。
- 知识图谱抽取：风险较高，因为三元组关系依赖句子和段落顺序。
- 论文引用：必须绑定原 PDF 页码和 evidence，重要结论人工复核。

## 修复方案

后续如果要把 OCR 作为知识图谱或高质量 RAG 语料，需要重跑一版 layout-aware OCR：

1. 保存每一行的坐标框。
2. 检测页面是否为两栏。
3. 两栏页按“左栏从上到下，再右栏从上到下”重排。
4. 表格、公式、图注页标记为高风险。
5. chunk metadata 保留页码、栏号、坐标和 layout 风险。
