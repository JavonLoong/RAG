# 项目成果总览

这个文件夹是给人看的入口，不是给程序看的底层目录。你平时先看这里，不用直接面对 `build_summary.json`、`runtime`、`chroma` 这些工程名。

这里有两类东西：

- 几 KB 的说明文件：告诉你“这是什么、怎么看、怎么汇报”。
- 名字里带 `真实文件夹` 的目录入口：点进去就是实际大文件或真实产物，不是空壳。

## 阅读顺序

1. `01_资料输入_14本PDF和问答JSON`
   老师资料和原始问答数据从哪里来。

2. `02_OCR结果_13本扫描PDF`
   13 本扫描版 PDF 的 OCR 结果和总表。

3. `03_普通RAG数据库_14本资料`
   13 本 OCR 文本 + 1 本可直接抽文本资料 + 问答 JSON，合成后的普通 RAG 可检索库。

4. `04_检索测试结果_10个问题`
   对数据库做的 10 个小问题检索测试。

5. `05_知识图谱POC_三元组和人工判断`
   JSON 到三元组、evidence、人工判断、知识图谱图片的流程。

6. `06_汇报材料_发群和组会`
   可以拿去发群或组会讲的材料。

## 重要约定

数据库本体暂时不能改成中文路径。原因是 ChromaDB 底层的 `hnswlib` 在 Windows 中文路径下会写不完整索引文件，导致数据库重启后打不开。所以数据库实际仍放在稳定的英文工程路径里，这里只做清晰入口和说明。
