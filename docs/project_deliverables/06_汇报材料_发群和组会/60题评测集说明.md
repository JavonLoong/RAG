# 60题评测集说明

## 目标

这套评测集用于把 RAG/GraphRAG 项目从“能演示”推进到“能评测、能复现、能解释”。它不追求题量堆砌，而是覆盖结项和挑战杯评审最关心的几类能力：

- 普通 RAG 能不能从动力装备资料中找回事实证据。
- 检索结果能不能支撑带依据的回答。
- OCR、JSON 入库和资料质量风险是否被识别。
- 知识图谱 POC 是否能用 evidence 和人工评审证明链路走通。
- GraphRAG context/global 场景是否有固定同题子集。
- 评测指标是否能支撑后续对比实验和失败归因。

机器可读数据集位置：

`evaluation/system_eval_questions.jsonl`

## 字段说明

每行是一条 JSON 记录，核心字段包括：

- `id`：题目编号。
- `question`：评测问题。
- `reference_answer`：人工标准答案，用于汇报和后续人工评分。
- `expected_evidence_keywords`：检索证据中应覆盖的关键词。
- `task_type`：题目类型，用于分组统计。
- `source_scope`：预期资料来源范围。
- `expected_modes`：适合的检索/回答模式，例如 `keyword`、`hybrid_rrf`、`graphrag_context`、`graphrag_global`。
- `grading_notes`：人工评分说明。

## 汇报口径

可以这样讲：

> 为了避免只做 demo，我整理了一套 60 题系统评测集。题目覆盖事实问答、流程理解、检索策略对比、OCR 风险、知识图谱 POC、GraphRAG 同题子集和评测方法。每道题都有标准答案、预期证据关键词和推荐模式，后续可以用统一脚本统计 retrieval recall、关键词覆盖率、证据缺失和失败类型。

## 下一步
