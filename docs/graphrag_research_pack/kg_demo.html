Three Tool Lines, Three Validation Goals

三条线不是排名，而是三种验证视角。

同一段燃气轮机/设备故障样例，可以用 Microsoft GraphRAG 看完整方法流程，用 Neo4j GraphRAG 看图数据库落地，用 LlamaIndex/LangChain 看如何快速接入已有 RAG pipeline。这里不下“谁最好”的结论，只说明各自适合验证什么。

选择工具线

当前 POC 是 rule-based/manual baseline，没有调用 LLM。这里展示的是后续可验证路线，不是已完成的自动抽取结果。

            检修记录 D2
            检查发现进气滤网堵塞，压气机叶片表面积灰。清理滤网并安排叶片离线清洗后，温度恢复正常。
          
      
          
      
            技术手册 D3
            压气机出口温度偏高常见原因包括进气阻力增大、叶片污染和温度传感器漂移。建议检查滤网、清洗叶片并校验传感器。
          
      
        
      
      
      

      
      
        
      三条线的验证边界
      
        
      
          
            
      
              路线
              它更像什么
              本阶段看什么
            
      
                
          
            
      
              Microsoft GraphRAG
              完整 GraphRAG 方法流程
              实体/关系/社区摘要是否能支持全局和局部问题。
            
      
            
      
              Neo4j GraphRAG
              图数据库工程落地
              schema 是否清晰，路径查询和可视化是否方便人工检查。
            
      
            
      
              LlamaIndex / LangChain
              现有 RAG 项目的快速集成层
              能否低成本抽三元组/属性图，并与向量检索一起召回上下文。
