## 组会怎么讲？（话术参考）

### 30秒版本

> "我做的是 RAG 管线的后端部分。
> 把 Label Studio 标注好的 JSON 数据，经过 **解析→清洗→分块→向量化** 四个步骤，
> 存到 ChromaDB 向量数据库里。
> 然后用 **语义检索 + BM25 关键词混合检索** 从知识库里找到最相关的内容。
> 前端有管理面板可以直接操作。"

### 2分钟版本

> "我的工作分四个部分：
>
> **第一，数据解析**。Label Studio 导出的 JSON 文件，每个标注框有坐标、标签(Title/Para/List)和文本内容。
> 我按坐标排序，把它们还原成有结构的文档。
>
> **第二，智能分块**。不是简单按字数切，而是碰到标题(Title)自动换一块，
> 长文本按句号、问号这些语义边界来切。这样每个 chunk 在语义上是完整的。
>
> **第三，向量化存储**。用 BGE-m3 模型把文本变成 1024 维的向量，存到 ChromaDB。
>
> **第四，混合检索**。同时用语义相似度和 BM25 关键词匹配两种方式检索，
> 最后用 RRF 算法把两种结果融合排序。这样既能理解语义，又不会漏掉精确关键词。"

## 每个模块做了什么（大白话）

### parsing.py — "翻译官"
把 Label Studio 导出的复杂 JSON 翻译成简单的文本块列表。
就像把一堆乱七八糟的标注框，按位置排好序，变成一篇能读的文章。

### cleaning.py — "清洁工"  
OCR有时候会把一段话切碎，比如 "程体系。" "建方法" 这种两三个字的碎片。
这个模块把碎片自动粘回去。

### chunking.py — "切蛋糕的"
把长文章切成 500 字左右的小块（chunk），但不是无脑切：
- 碰到**标题**就换一块（保证标题和内容在一起）
- 切的时候优先在**句号**的地方切（不把句子切断）
- 相邻两块有 50 字的重叠（保证上下文连续）

### embeddings.py — "翻译成数学语言"
用 BGE-m3 模型把文字变成 1024 个数字组成的向量。
语义相近的文字，向量也相近（用余弦距离衡量）。

### retrieval.py — "双管齐下找答案"
两种搜索方式同时用：
1. **语义检索**：理解你的意思去找（"发动机故障"也能找到"引擎异常"）
2. **BM25 关键词**：精确匹配关键词（搜 "QC185" 就一定能找到含 "QC185" 的段落）
最后用 RRF 算法把两种结果合并排序。

### pipeline.py — "总指挥"
把上面所有步骤串起来：解析 → 清洗 → 分块 → 向量化 → 存库。
一个命令搞定全部流程。
