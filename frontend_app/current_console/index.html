                GraphRAG / Query Plan

              


              


                


                  01 INPUT

                  提问与参数

                  
读取问题、集合名、Graph DB、top_k=100、检索模式和 LLM 配置。


                


                


                


                  02 ROUTER

                  Query Router / 问题分流

                  
判断是向量问答、局部实体追问、全局汇总，还是需要全量逐联系人分析。


                


                


                


                  03 SOURCES

                  证据源选择

                  
读取微信文本集合、GraphStore、社区摘要；普通 top-k 只作补充，不作为全量结论依据。


                


                


                


                  04 FULL SCAN

                  全量逐联系人扫描

                  
枚举每个一对一联系人，排除群聊、公众号、系统通知、文件传输助手后扫描全部消息。


                


                


                


                  05 EVIDENCE

                  原文证据审计

                  
只保留带日期、发送方、chunk_id 或 message_id 的原文证据；无原文则只能标为证据不足。


                


                


                


                  06 OUTPUT

                  LLM 判级与汇总

                  
按联系人输出较确定、疑似、性别不确定或不足证据，并说明为什么其他联系人未计入。


                


              


              


                


                  VECTOR_ONLY: 普通语义片段问答

                  LOCAL_SEARCH: 单个实体邻域追问

                  GLOBAL_SEARCH: 跨社区/全局总结

                  CONTACT_FULL_SCAN: 全量私聊联系人级分析
