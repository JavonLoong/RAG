完整评测流程

GitHub Pages 不能直接启动 Python、ChromaDB 或本地后端，所以这个页面做的是浏览器端仓库级评测。真实在线检索压测可以作为下一阶段接后端 API。

01导入仓库解析 GitHub URL，读取指定分支的完整文件树。
02识别能力模块扫描 data_pipeline、retrieval_engine、rag_orchestrator、kg_pipeline、evaluation、tests 等模块。
03解析评测资产读取问题集、baseline 对比、失败分析、GraphRAG 子集报告。
04评分与建议按结构、数据集、baseline、失败归因、GraphRAG、复现性给分。
