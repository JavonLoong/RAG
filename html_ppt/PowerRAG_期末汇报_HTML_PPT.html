PowerRAG02 / 20

这不是一个普通问答页面，而是一条从资料到可信答案的工程线。

汇报围绕四类能力展开：资料问答、关系追踪、综合能力、评测能力。

01

资料问答

PDF、扫描页、表格、JSON 标注和维修报告统一进入系统。

02

关系追踪

部件、故障、工况、参数和维修动作通过实体关系连接。

03

综合能力

跨文档、跨部件问题不只返回单段文本，而是汇总证据。

04

评测能力

用 baseline、公开 benchmark 和失败归因判断系统路线。

动力装备领域资料散落在技术手册、检修规程、历史故障案例、行业标准和工程师经验中。

PowerRAG05 / 20
初心

可信答案不是一句话，而是问题、答案、证据的组合。

PowerRAG 的价值不在“回答流畅”，而在答案能回到原始资料，被检查、被引用、被继续优化。

示例问题

燃气轮机某类故障可能和哪些部件或工况有关？

答案可能关联压气机、燃烧室、负荷变化与参数异常。
证据维修资料中的故障描述、检查步骤和处理措施。原文
来源文件名、页码、标题、段落和上下文。可追溯

不同资料必须走不同通道，最后进入同一套后续流程。

JSON、文字型 PDF、扫描型 PDF 的处理方式不同；统一入口是后续检索、图谱和评测的前提。

JSON 标注文件名、页码、框选区域、类别结构化字段
文字 PDF直接抽取页面文字与标题文本证据

PowerRAG08 / 20

真实资料没有那么干净：OCR 是资料入口工程的一部分。

真实问题

  • 扫描页、图片页、印章和页眉页脚
  • 表格、图像、不规则排版
  • 没有 OCR，系统看不到文字

方案试错

  • 开源 OCR 路线
  • 云 OCR 路线
  • 对中文、表格、版面保持差异显著

形成判断

RAG 的上限不只取决于模型，也取决于资料进入系统之前有没有被正确处理。

PowerRAG10 / 20

证据进入三个空间：关键词、向量、图谱，各自服务不同问题。

关键词空间

BM25 / keyword 对型号、故障名、部件名等强术语敏感。

向量空间

把文本转成高维语义表示，寻找意思相近的证据。

图谱空间

保存部件、故障、工况、参数、维修动作之间的关系。

全局问题

使用社区摘要和全局检索，回答跨文档规律。

PowerRAG20 / 20
总结

PowerRAG 的核心不是“做了一个网页”，而是把真实资料变成可信知识系统。

资料进入系统，形成可引用证据；证据进入关键词、向量和图谱空间；评测平台再用题集、baseline 和失败归因验证效果。

下一阶段

  • 引入 reranker，改善召回后的排序。
  • 对 GraphRAG local / global 做同题实测。
  • 扩展中文动力装备 benchmark 与专家评分。
