# 公开书籍标注反馈

## 本次检测结论

- 这批 JSON 是连续导出的快照，后一个基本包含前一个；入库只用最新快照，不能 27 个都入库。
- 最新快照里有 3961 页任务，抽出 54443 个文本块。
- 目前可直接作为正文候选的文本块约 26000 条，其余需要作为元数据、人工复核或过滤掉。
- 最大问题不是 JSON 不能解析，而是很多 `Para` 实际是按行切开的文本，入库前必须按页面坐标合并成更完整的段落。

## 标注人员需要注意

1. `Para` 只标正文，不要标页码、页眉、目录碎片、审核栏、批准栏。
2. `Title` 只标章节标题或小节标题，后续主要做 metadata。
3. `Figure` 只标图题或图注，不要框整张图。
4. `Table` 只保留有故障、机理、参数价值的表格；复杂表格需要人工复核。
5. `Formula` 只标公式本体，公式解释应标成 `Para`。
6. `List` 可以作为正文候选，但不能是目录、编号清单或空泛条目。
7. OCR 明显错字、乱码、左右栏串行时，需要标记为待修正，不能直接入库。

## 入库前筛选规则

- 直接入库：`Para` / 有内容的 `List`，长度不少于 20 字，非重复页眉页脚，符号噪声不高。
- 只做元数据：短 `Title`、普通图片标题。
- 人工复核：`Table`、`Formula`、图注、含领域词的短句、符号比例偏高文本。
- 不入库：空文本、页码、过短碎片、高频重复页眉页脚、无领域信息的表格/图注/公式。

## 后续工程动作

1. 用最新 JSON 快照解析出文本块。
2. 按上述规则筛选。
3. 同一页内按坐标顺序合并相邻 `Para` / `List`。
4. 合并后的正文入 ChromaDB。
5. `Table` / `Figure` / `Formula` 进入人工复核队列。
